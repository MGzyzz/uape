# Документация проекта UAPE

Этот документ описывает всё, что было сделано в проекте: что это такое, как устроено, какие функции реализованы и как они работают.

---

## Что такое UAPE

UAPE — образовательная веб-платформа для обучения программированию. Суть платформы:

1. Пользователь регистрируется и проходит **опрос** (онбординг): выбирает свою область интересов (например, веб-разработка, искусственный интеллект) и профессию
2. На основе этих данных платформа **подбирает персональные разделы** с курсами и каналами
3. Пользователь может **добавлять материалы в закладки** и переходить на YouTube напрямую

Весь контент — это реальные YouTube-каналы и плейлисты. Администратор добавляет их вручную (вставляет ссылку), а платформа автоматически загружает название, обложку, количество видео и подписчиков через официальный YouTube API.

---

## Из чего состоит проект

Проект разделён на две части, которые работают вместе:

### Бэкенд (сервер)
- Хранит все данные (пользователи, курсы, каналы, плейлисты)
- Обрабатывает регистрацию и вход
- Общается с YouTube API для получения данных
- Отвечает на запросы фронтенда

**Технологии:** Python 3.11, Django 5.2, SQLite (база данных), JWT-авторизация

### Фронтенд (интерфейс)
- То, что видит пользователь в браузере
- Отображает курсы, карусели, профиль, онбординг
- Обращается к серверу для получения данных

**Технологии:** React 19, Vite, Tailwind CSS

## Что было реализовано

### Авторизация и пользователи

- **Регистрация** — создание аккаунта с именем, фамилией, email и паролем
- **Вход** — авторизация по email и паролю
- **JWT-токены** — безопасный механизм авторизации. При входе выдаётся два токена:
  - `access` — действует 60 минут, используется в каждом запросе
  - `refresh` — действует 7 дней, автоматически обновляет access-токен
- **Профиль пользователя** — страница с аватаром, именем, информацией об аккаунте

### Онбординг (опрос при регистрации)

Трёхшаговый опрос для персонализации контента:

- **Шаг 1 — Область интересов:** Веб-разработка, ИИ, Мобильная разработка, Кибербезопасность и др. (11 вариантов)
- **Шаг 2 — Профессия:** Студент, Специалист, Карьерный переход и др.
- **Шаг 3 — Навыки:** Многовыборный список технологий (Python, JavaScript, Java и др.)

Данные сохраняются в модели `UserOnboarding`. На их основе фильтруются разделы на главной.

### Контент (курсы, каналы, плейлисты)

Хранится в базе данных и управляется через панель администратора.

**Модели данных:**

| Модель | Что хранит |
|--------|-----------|
| `Channel` | YouTube-канал: название, аватар, подписчики, описание, ссылка |
| `Playlist` | YouTube-плейлист: название, обложка, количество видео, канал |
| `Video` | YouTube-видео: название, обложка, длительность, канал |
| `Tag` | Тег с цветом (например: #python, #javascript) |
| `Section` | Раздел на главной странице: название, подзаголовок, тип контента, привязанное поле онбординга |
| `Bookmark` | Закладка пользователя на плейлист, видео или канал |
| `UserOnboarding` | Данные опроса пользователя |

**Как добавляется контент:**
1. Администратор вставляет YouTube URL в панели управления
2. Система автоматически запрашивает YouTube Data API v3 и сохраняет данные
3. Обновление происходит при каждом сохранении в админке

**Теги** привязаны к плейлистам и каналам. Текущие теги:
- `#python` — синий (#3776AB)
- `#javascript` — жёлтый (#F7DF1E)
- `#java` — оранжевый (#ED8B00)
- `#c++` — голубой (#00599C)
- `#c#` — зелёный (#239120)

### Главная страница — разделы и карусели

Разделы на главной странице (`/profile`) динамические — формируются под конкретного пользователя:

- Если пользователь выбрал в онбординге область "Веб-разработка", ему показывают разделы с тегом `web` (или соответствующим полем)
- **Персональные рекомендации** — раздел "Recommended for you" формируется по навыкам пользователя: его теги из навыков совпадают с тегами плейлистов
- Разделы с плейлистами/видео отображаются в виде **карусели** (листается стрелками)
- Разделы с каналами — в виде **сетки**

**Текущее наполнение базы данных:**
- 6 разделов с плейлистами (Python, JavaScript, Java, C++, C#, Recommended)
- 11 разделов с каналами (по одному на каждую область онбординга)
- 16 плейлистов
- 21 канал (Traversy Media, Net Ninja, Fireship, freeCodeCamp и другие)

### Закладки

- Авторизованный пользователь может добавить любой плейлист, видео или канал в закладки
- Значок "сердечко" на карточке — нажатие добавляет/убирает из закладок
- Данные сохраняются на сервере, синхронизированы между устройствами

### Загрузка и скелетоны

Пока данные грузятся с сервера, пользователь видит анимированные "скелеты" — серые мерцающие заглушки в форме карточек. Это предотвращает "прыжки" контента и даёт понять, что страница загружается.

Каждое изображение тоже загружается плавно: пока картинка не загрузилась — видна анимированная заглушка, затем изображение появляется с плавным переходом.

## API (программный интерфейс)

Все данные передаются между фронтендом и бэкендом через API. Вот основные адреса:

| Метод | Адрес | Что делает |
|-------|-------|-----------|
| POST | `/api/auth/register/` | Регистрация |
| POST | `/api/auth/login/` | Вход, возвращает токены |
| POST | `/api/token/refresh/` | Обновление access-токена |
| GET | `/api/profile/` | Данные профиля текущего пользователя |
| GET/PATCH | `/api/onboarding/` | Чтение и сохранение данных опроса |
| GET | `/api/courses/sections/` | Разделы главной страницы (с фильтрацией по пользователю) |
| GET | `/api/courses/recommended/` | Персональные рекомендации по навыкам |
| GET | `/api/courses/playlists/` | Список всех плейлистов |
| GET | `/api/courses/videos/` | Список всех видео |
| GET | `/api/courses/channels/` | Список всех каналов |
| POST | `/api/courses/bookmarks/` | Добавить в закладки |
| DELETE | `/api/courses/bookmarks/` | Убрать из закладок |

## Структура папок

```
uape/
├── back_end/              # Сервер (Python / Django)
│   ├── accounts/          # Пользователи, авторизация, онбординг

└── front_end/             # Интерфейс (React)
    └── src/
        ├── api/           # Функции для обращения к серверу
        ├── app/           # Роутинг (какой URL → какая страница)
        ├── pages/
        │   ├── auth/      # Страница входа/регистрации
        │   ├── landing/   # Главная (лендинг) страница
        │   ├── profile/   # Страница пользователя с курсами
        │   └── onboarding/# Страница опроса
        └── shared/        # Общие компоненты, иконки, шапка, подвал
```

## Настройки и переменные окружения

### Бэкенд (`back_end/.env`)

| Переменная | Описание |
|-----------|----------|
| `YOUTUBE_API_KEY` | Ключ YouTube Data API v3. Получается в Google Cloud Console |

### Фронтенд (`front_end/.env`)

| Переменная | Описание |
|-----------|----------|
| `VITE_API_URL` | Адрес API. Значение `/api` — работает через встроенный прокси |
| `VITE_API_TARGET` | Куда проксировать запросы. По умолчанию `http://localhost:8000` |
| `NGROK_HOST` | Домен ngrok для тестирования с телефона или внешнего устройства |

## Что ещё планируется

- **Тест оценки уровня** — пользователь проходит тест, по результатам определяются его сильные и слабые стороны, разделы подстраиваются под них
- **Детальная страница курса** — отдельная страница плейлиста с описанием, информацией о канале и блоком рекомендаций
- **Редирект после регистрации** — новый пользователь сразу попадает на онбординг
